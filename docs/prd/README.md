# Product Requirements Documents (PRD)

신규 기능/제품 요구사항의 ***무엇을, 누구를 위해, 왜*** 를 남기는 곳. 한 기능 = 한 파일.

- **템플릿**: `.claude/templates/prd-template.md`
- **스캐폴딩**: `/doc prd <제목>` (worktree 안에서 다음 번호로 생성)
- **파일명**: `NNNN-<slug>.md` (4자리 zero-pad, 소문자 하이픈 slug). 예: `0001-order-bulk-import.md`
- **번호**: 단조 증가, 재사용 금지. 드롭된 기능도 파일을 **삭제하지 말고** 상태를 `Dropped`로.
- **PRD vs ADR**: *무엇/누구를 위해* 는 PRD, *어떻게/왜 그 선택* 은 ADR. 판단 기준은 `.claude/skills/document-first/SKILL.md`.
